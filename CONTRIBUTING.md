# Contributing

## Contribution Types
- RFC clarifications
- architecture notes
- spec draft proposals
- validation tooling improvements

## Pull Request Checklist
- [ ] Changes are scoped and well-justified
- [ ] Relevant docs are updated
- [ ] `python scripts/validate_repo.py` passes
- [ ] No secrets or generated artifacts are committed

## Spec Change Rules
For normative changes, include:
1. rationale
2. invariant impact
3. migration impact (if any)
4. at least one invalid example
