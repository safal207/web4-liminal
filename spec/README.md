# Spec Roadmap

This directory hosts **normative** protocol specifications for the Web4 Liminal Care Stack.

Current status: incubation.

## Planned Spec Surface

- `capu.md` - Causal Processing Unit normative behavior
- `cml.md` - Causal Memory Layer normative structure
- `lpt.md` - Liminal Passage constraints and transition semantics
- `t-trace.md` - Irreversible transition trace format/invariants
- `rinse.md` - Reflective integration model

## Versioning Policy

- `v0.x` - exploratory, backward compatibility not guaranteed
- `v1.0` - stable normative baseline
- all breaking changes require explicit migration notes

## Conformance Model

Each spec should define:

1. required fields and structures
2. mandatory invariants
3. invalid-state examples
4. reference validation procedure
5. minimal canonical fixtures
